# Editing Files on GitHub (Quick Guide)

A simple, step-by-step guide for making edits to files in a GitHub repository using your browser—no technical experience required.

---

## Overview

You can edit files directly on GitHub without installing anything. Just open a file, click edit, make your changes, and save.

---

## Steps

### 1. Open the repository and find your file

Go to the GitHub project page and browse the file list. Click the file you want to edit.

**You’ll see:**
A preview of the file contents.

---

### 2. Click the ✏️ Edit button

In the top-right corner of the file view, click the **pencil icon**.

```
✏️ Edit this file
```

---

### 3. Make your changes

Edit the file directly in your browser.

**You’ll see:**
A text editor with the file content.

---

### 4. Add a commit message

Scroll to the bottom and describe your change.

**Example:**

```
Fix typo in introduction
```

---

### 5. Save your changes

Click the green button:

* **Commit changes**

---

### 6. Don’t have permission?

GitHub will help you automatically:

* Creates your own copy (*fork*)
* Lets you submit a *pull request* for review

**You’ll see:**
**Propose changes** instead of “Commit changes”

---

## ✅ Quick Summary

```
Open file → Click ✏️ → Edit → Add message → Commit changes
```
